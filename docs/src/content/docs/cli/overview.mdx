:::tip[Recommended: use the MCP server]
For AI coding assistants (Claude Code, Cursor, Windsurf, and more), the recommended way to use Lucerna is via its built-in **MCP server** — it indexes your project automatically and exposes `search_codebase` and `get_neighbors` tools directly to your agent. See the [Quickstart](/quickstart) to get started.
:::

The `lucerna` CLI lets you index, search, and inspect codebases directly from the terminal — no code required.

## Running without installing

Use a package manager's runner to execute Lucerna as a one-off command:

```bash
npx -y @upstart.gg/lucerna@latest <command>
# or
pnpm dlx @upstart.gg/lucerna@latest <command>
# or
bunx @upstart.gg/lucerna@latest <command>
```

## Installing globally

For repeated use, install Lucerna globally so the `lucerna` binary is always available:

```bash
npm install -g @upstart.gg/lucerna
# or
pnpm add -g @upstart.gg/lucerna
# or
bun add -g @upstart.gg/lucerna
```

Then use it directly:

```bash
lucerna <command>
```

## Available commands

| Command | Description |
|---------|-------------|
| [`install`](/cli/install) | Interactive setup wizard (MCP registration + embedding config) |
| [`index`](/cli/index) | One-shot indexing of a project |
| [`watch`](/cli/watch) | Index and re-index on file changes |
| [`search`](/cli/search) | Search an existing index |
| [`graph`](/cli/graph) | Explore knowledge graph relationships |
| [`stats`](/cli/stats) | Show index statistics |
| [`clear`](/cli/clear) | Delete the index |
| [`eval`](/cli/eval) | Measure recall@k against a query set |
