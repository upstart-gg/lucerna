Lucerna supports multiple remote embedding and reranking providers. The recommended way to configure them is via your `.mcp.json` file — see [Env Vars](/configuration/env-vars) for the full variable reference, or [Custom Providers](/configuration/custom-providers) for the `lucerna.config.ts` approach.

:::note[Pricing]
Prices shown are approximate as of April 2026 and subject to change. Always verify at the provider's pricing page before budgeting.
:::

---

## Embedding providers

Each snippet below shows the `env` block to add to your `.mcp.json`:

```json title=".mcp.json"
{
  "mcpServers": {
    "lucerna": {
      "command": "npx",
      "args": ["-y", "@upstart.gg/lucerna", "mcp-server"],
      "env": {
        "LUCERNA_EMBEDDING": "voyage:voyage-code-3",
        "VOYAGE_API_KEY": "your-key"
      }
    }
  }
}
```

:::tip[Using unlisted models]
For models not in the tables below, pass dimensions as a third segment:

```json
{ "LUCERNA_EMBEDDING": "ollama:my-custom-model:768" }
```

OpenAI's `text-embedding-3` models also support server-side Matryoshka truncation — passing a smaller dimension produces shorter vectors directly from the API (better quality than post-hoc truncation):

```json
{ "LUCERNA_EMBEDDING": "openai:text-embedding-3-large:256" }
```
:::

### Voyage ⭐ recommended for code

Best-in-class for code retrieval. `voyage-code-3` is purpose-trained on code with a 32K context window.

```json title=".mcp.json → env"
{
  "LUCERNA_EMBEDDING": "voyage:voyage-code-3",
  "VOYAGE_API_KEY": "your-key"
}
```

| Model | Dimensions | Price per 1M tokens | Notes |
|-------|-----------|---------------------|-------|
| `voyage-code-3` | 1024 | $0.18 | Best for code, 32K ctx — **top pick** |
| `voyage-3` | 1024 | $0.06 | General text |
| `voyage-3-lite` | 512 | $0.02 | Faster, lower cost |

### GCP Vertex AI

## Reranking providers

Re-ranks top results from hybrid search using a cross-encoder, improving precision at the cost of additional latency.

---

### Voyage ⭐ recommended

```json title=".mcp.json → env"
{
  "LUCERNA_RERANKING": "voyage:rerank-2",
  "VOYAGE_API_KEY": "your-key"
}
```

| Model | Price per 1K queries | Notes |
|-------|---------------------|-------|
| `rerank-2` | $0.05 | Best quality, 16K ctx per pair |
| `rerank-2-lite` | $0.02 | Lower latency |

### GCP Vertex AI Ranking

Proper cross-encoder using Google's dedicated Ranking API. No data store or app needed — just a GCP project with the Discovery Engine API enabled.

:::note[One-time setup]
Enable the Discovery Engine API in your GCP project before first use:
```bash
gcloud services enable discoveryengine.googleapis.com
```
:::

```json title=".mcp.json → env"
{
  "LUCERNA_RERANKING": "vertex:semantic-ranker-default-004",
  "GOOGLE_CLOUD_PROJECT": "my-gcp-project",
  "GOOGLE_ACCESS_TOKEN": "obtain via: gcloud auth print-access-token"
}
```

| Model | Price per 1K queries | Notes |
|-------|---------------------|-------|
| `semantic-ranker-default-004` | $1.00 | Recommended, 1024 token ctx per doc |
| `semantic-ranker-fast-004` | $1.00 | Lower latency |

1 query = up to 100 documents; every additional 100 docs counts as +1 query.

### Gemini

Prompt-based relevance scoring using a Gemini generative model. Not a cross-encoder — uses `GOOGLE_API_KEY`, no GCP project required.

```json title=".mcp.json → env"
{
  "LUCERNA_RERANKING": "gemini:gemini-2.0-flash-lite",
  "GOOGLE_API_KEY": "your-key"
}
```

| Model | Price per 100 docs | Notes |
|-------|-------------------|-------|
| `gemini-2.0-flash-lite` | ~$0.0002 | Cheapest; good enough for most use cases |
