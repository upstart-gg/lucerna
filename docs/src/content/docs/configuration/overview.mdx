---
title: Configuration
description: Configure embedding and reranking providers via env vars or lucerna.config.ts.
---

Lucerna requires an embedding provider to enable semantic (vector) search. Without one, only lexical (BM25) search is available.

There are two ways to configure providers:

1. **[Environment variables](/configuration/env-vars)** *(recommended)* — set `LUCERNA_EMBEDDING=provider:model` in your MCP server config. No files to add to your project, works across all AI clients.
2. **[Config file](/configuration/config-file)** — place a `lucerna.config.ts` at the project root. Useful when you need programmatic control or a [custom provider](/configuration/custom-providers).

:::tip[Prefer env vars for MCP]
Setting `LUCERNA_EMBEDDING` and the provider API key directly in your MCP server config keeps configuration out of your repo and works identically across all AI coding clients.
:::

## Priority order

When multiple configuration sources are present, the highest-priority source wins:

| Priority | Source |
|----------|--------|
| 1 (highest) | `--no-semantic` flag (disables embeddings entirely) |
| 2 | `--embedder` / `--reranker` CLI flags |
| 3 | `lucerna.config.ts` / `.js` |
| 4 (lowest) | `LUCERNA_EMBEDDING` / `LUCERNA_RERANKING` env vars |

:::tip[Reranking is optional]
Without a `rerankingFunction`, hybrid search still applies **RRF fusion** automatically — merging vector and full-text results into a single ranked list. This is a solid baseline for most codebases. Add a [reranking provider](/configuration/reranking-providers) when you need higher precision on large codebases where many chunks look similar at the embedding level.
:::
