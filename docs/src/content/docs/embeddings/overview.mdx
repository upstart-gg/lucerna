---
title: Embeddings Overview
description: How Lucerna selects and uses embedding models.
---

Embeddings power the semantic (vector) search component of Lucerna's hybrid search. When `embeddingFunction` is not set, the indexer auto-selects:

1. **`CloudflareEmbeddings`** — if `CLOUDFLARE_ACCOUNT_ID` and `CLOUDFLARE_API_TOKEN` are set in the environment
2. **`JinaCodeEmbeddings`** — otherwise, runs a local code-optimised model via ONNX (no API key required)

## Options

| Option | Description |
|--------|-------------|
| [Local models](/embeddings/local) | `JinaCodeEmbeddings`, `BGESmallEmbeddings`, `HFEmbeddings` — run locally via ONNX |
| [Cloudflare](/embeddings/cloudflare) | `CloudflareEmbeddings` — remote, uses Workers AI |
| [Custom](/embeddings/custom) | Implement `EmbeddingFunction` to use any provider |
| `false` | Disable semantic search entirely — BM25 only, no model download |

## Disable semantic search

```ts
const indexer = new CodeIndexer({
  projectRoot: '.',
  embeddingFunction: false,
});
```
