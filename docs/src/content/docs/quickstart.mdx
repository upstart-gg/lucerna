Lucerna ships a built-in [Model Context Protocol](https://modelcontextprotocol.io/) server so AI coding assistants (Claude Code, Cursor, Windsurf, Zed, VS Code, and more) can search your codebase directly.

## Setup

Run the interactive installer from your project directory:

<Tabs>
  <TabItem label="npx">
    ```bash
    npx -y @upstart.gg/lucerna@latest install
    ```
  </TabItem>
  <TabItem label="bunx">
    ```bash
    bunx @upstart.gg/lucerna@latest install
    ```
  </TabItem>
  <TabItem label="pnpx">
    ```bash
    pnpm dlx @upstart.gg/lucerna@latest install
    ```
  </TabItem>
  <TabItem label="yarn dlx">
    ```bash
    yarn dlx @upstart.gg/lucerna@latest install
    ```
  </TabItem>
</Tabs>

The installer will:

1. **Detect and configure your AI clients** — registers the Lucerna MCP server with every AI coding assistant found on your machine (Claude Code, Cursor, Windsurf, Zed, VS Code, and [more](https://github.com/neondatabase/add-mcp))
2. **Scaffold `lucerna.config.ts`** — creates a config file at your project root if one doesn't exist yet (edit it afterwards to configure an embedding provider for semantic search)
3. **Update `.gitignore`** — appends `.lucerna/` so the index directory is never committed

Restart your AI client after running the installer to load the MCP server.

:::tip[Manual setup]
Prefer to configure things yourself? Use [`add-mcp`](https://github.com/neondatabase/add-mcp) directly to register the MCP server, then create a `lucerna.config.ts` manually. See [Config File](/configuration/config-file) for the full reference.

```bash
npx -y add-mcp "npx -y @upstart.gg/lucerna@latest mcp-server" --name lucerna
```

Also add `.lucerna/` to your `.gitignore` so the index directory is never committed:

```
# .gitignore
.lucerna/
```
:::

See [MCP Tools](/mcp-tools) for the full tool reference.
