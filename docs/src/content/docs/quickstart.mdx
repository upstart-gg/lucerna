Lucerna ships a built-in [Model Context Protocol](https://modelcontextprotocol.io/) server so AI coding assistants (Claude Code, Cursor, Windsurf, Zed, VS Code, and more) can search your codebase directly.

## Setup

### 1. Add Lucerna to your AI coding assistant

Use [`add-mcp`](https://github.com/neondatabase/add-mcp), which supports 14 clients and writes the correct config file automatically:

<Tabs>
  <TabItem label="npx">
    ```bash
    npx -y add-mcp "npx -y @upstart.gg/lucerna@latest mcp-server" --name lucerna
    ```
  </TabItem>
  <TabItem label="bunx">
    ```bash
    bunx add-mcp "bunx @upstart.gg/lucerna@latest mcp-server" --name lucerna
    ```
  </TabItem>
  <TabItem label="pnpx">
    ```bash
    pnpx add-mcp "pnpx @upstart.gg/lucerna@latest mcp-server" --name lucerna
    ```
  </TabItem>
  <TabItem label="yarn dlx">
    ```bash
    yarn dlx add-mcp "yarn dlx @upstart.gg/lucerna@latest mcp-server" --name lucerna
    ```
  </TabItem>
</Tabs>

`add-mcp` will prompt you to choose which clients to configure. The MCP server uses **the working directory of the client** as the project root — no hardcoded paths needed.

### 2. Ignore the index directory

Lucerna stores its index (vector database, file hashes) in a `.lucerna` directory at your project root. Add it to your `.gitignore`:

```
# .gitignore
.lucerna/
```

Or, if you work across many projects, add it once to your global gitignore (`~/.gitignore_global`) instead.

### 3. Enable semantic search *(optional)*

:::caution[Semantic search requires an embedding provider]
Without a provider configured, Lucerna falls back to **lexical (BM25) search only** — and the MCP tools will include a warning in their output. For best results, configure an embedding provider.
:::

The recommended approach is to add env vars to your MCP server config:

```json
{
  "mcpServers": {
    "lucerna": {
      "command": "npx",
      "args": ["@upstart.gg/lucerna@latest", "mcp-server"],
      "env": {
        "LUCERNA_EMBEDDING": "voyage:voyage-4",
        "VOYAGE_API_KEY": "your-voyage-api-key"
      }
    }
  }
}
```

Or place a `lucerna.config.ts` at the project root — the MCP server picks it up automatically:

```ts
// lucerna.config.ts
import { VoyageEmbeddings } from '@upstart.gg/lucerna';

export default {
  embeddingFunction: new VoyageEmbeddings({ model: 'voyage-4' }),
};
```

See [Embedding Providers](/configuration/embedding-providers) for all available providers and recommended models, [Reranking Providers](/configuration/reranking-providers) for optional cross-encoder reranking, and [Env Vars](/configuration/env-vars) for the full variable reference.

See [MCP Tools](/mcp-tools) for the full tool reference.
